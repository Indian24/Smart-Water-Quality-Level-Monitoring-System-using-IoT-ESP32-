# Contributing

Thank you for your interest in contributing to this project!

## How to contribute

1. Fork the repository.
2. Create a branch:
   - `feature/<short-description>` for new features
   - `fix/<short-description>` for bug fixes
3. Implement changes and add tests or documentation if applicable.
4. Commit with clear messages (use Conventional Commits style).
5. Open a Pull Request to `main` and fill the PR template.

## Commit message format
- `feat: ...`
- `fix: ...`
- `docs: ...`
- `chore: ...`

## Coding style
- Keep functions small and single-purpose.
- Add comments for complex logic.
- Avoid hard-coded secrets; use `secrets.h`.

## Review process
- PRs will be reviewed by maintainers.
- Address review comments and keep commits clean (squash if needed).
